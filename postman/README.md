## Setup

1. **Import** the collection in Postman (File ‚Üí Import ‚Üí select `api_bloxfi.postman_collection.json`).
2. **Set collection variables:**
   - `baseUrl` ‚Äî e.g. `http://localhost:3000` (default).
   - `apiKey` ‚Äî Bearer token, **exactly 32 alphanumeric characters**. Create via your app (e.g. `npm run create-api-key`) or use a test key from env.
   - `requestId` ‚Äî Auto-generated by the **Pre-request Script** on every request that needs it (Create User, Update KYB, Submit KYB, Create Account, Add Wallet, Create Onramp, Create Offramp, Create High-Value Request). You don‚Äôt need to set or change it manually.

3. **Auto-saved variables** (set by test scripts when you run requests in order):
   - `userId` ‚Äî from Create Business User
   - `fileId` ‚Äî from Upload File
   - `accountId` ‚Äî from Create Account (onramp or offramp)
   - `walletId` ‚Äî from Add External Wallet
   - `onrampId` ‚Äî from Create Onramp
   - `offrampId` ‚Äî from Create Offramp
   - `highValueRequestId` ‚Äî from Create High-Value Request

   You can also set these manually from response bodies if you run requests out of order.

## E2E testing flow

1. **Health check** ‚Äî confirm API is up (no auth).
2. **Create Business User** ‚Äî `userId` is auto-saved.
3. **Get User** (optional) ‚Äî verify user.
4. **Update KYB Information** ‚Äî use a **new** `requestId` in header.
5. **Upload File** ‚Äî select a PDF/JPEG/PNG (max 10MB); `fileId` is auto-saved.
6. **Attach KYB Documents** ‚Äî body uses `{{fileId}}` and `{{userId}}`.
7. **Submit KYB Application** ‚Äî new `requestId`.
8. **Get KYB Status** (optional).
9. **Create Account (onramp)** ‚Äî new `requestId`; user must be KYB-approved for the rail; `accountId` is auto-saved.
10. **Create Account (offramp)** (optional) ‚Äî if you need a separate fiat account; new `requestId`.
11. **Add External Wallet** ‚Äî new `requestId`; `walletId` is auto-saved.
12. **Get Onramp Rates** (optional).
13. **Create Onramp** ‚Äî body uses `{{userId}}`, `{{accountId}}`, `{{walletId}}`; body `requestId` must match header; new `requestId`; `onrampId` is auto-saved.
14. **Get Onramp** / **List Onramps** (optional).
15. **Get Offramp Rates** (optional).
16. **Create Offramp** ‚Äî same IDs; new `requestId`; `offrampId` is auto-saved.
17. **Get Offramp** / **Cancel Offramp** (optional).
18. **Get Limits** ‚Äî platform limits.
19. **Get User Limits** ‚Äî user-effective limits (uses `{{userId}}`).
20. **Create High-Value Request** ‚Äî body `requestId` must match header; new `requestId`; `highValueRequestId` is auto-saved.
21. **Get High-Value Request** (optional).

**Important:** Each idempotent POST has a **Pre-request Script** that generates a new UUID and sets `requestId` before the request is sent, so you can run the flow repeatedly without 409 Conflict.

## Auth

The collection uses **Bearer token** auth at the collection level. Set `apiKey` to a valid 32-character API key. The app sends `Authorization: Bearer {{apiKey}}`.

## File upload

**Upload File** uses `multipart/form-data` with one field: key `file`, type **File**. In Postman: Body ‚Üí form-data ‚Üí key `file`, type **File**, then choose a PDF, JPEG, or PNG (max 10MB). After 201, `fileId` is saved automatically for **Attach KYB Documents**.
